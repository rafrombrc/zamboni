{% extends "ecosystem/design/xtag.html" %} {% block description %} An alert popup dialog is a small window which demands user input related to the current activity. It disables all current activities and only disappears after the user confirms an action or dismisses the dialog. {% endblock %} {% block use_it %} When the current activity absolutely requires user input, usually validation (e.g."delete all contacts?") or authorization (e.g. "enable GPS?"). {% endblock %} {% block tips %}

Do's

  • use action oriented questions or statements (e.g. "delete all contacts?").
  • use action labels such as "Save" instead of "OK" on the buttons.
  • short questions or statements
  • as a rule of thumb, place the primary button like "Save" on the right, and the secondary button like "Discard" or "Reset" on the left.
  • visually reduce the importance of secondary button like "Cancel" (e.g. use a gray background for the secondary button).

Dont's

  • don't use a title unless the action has severe consequences (e.g. Warning: "delete all contacts?").
  • don't use negation in your question or statement
  • don't use an alert when the action is trivial and reversible. Use a toast instead.
  • don't use an alert to prompt for make trivial choices. Consider a select list instead.
